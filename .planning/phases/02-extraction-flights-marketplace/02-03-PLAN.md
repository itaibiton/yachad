4. **Min seats**: Small number input or select (1-10). Label: `t("filterSeats")`.

**Part A: Replace `src/app/[locale]/(dashboard)/flights/page.tsx`**

Replace the existing "Coming Soon" placeholder with a Server Component that uses `preloadQuery` for SSR.

```typescript
// Server Component — async
import { Suspense } from "react";
import { FlightsClientPage } from "@/shared/components/flights/FlightsClientPage";
import { FlightCardSkeleton } from "@/shared/components/LoadingSkeleton";

export default async function FlightsPage() {
  // NOTE: preloadQuery does NOT support pagination. The client-side
  // usePaginatedQuery handles real-time data. This page simply renders
  // the client page which manages its own data fetching.
  // preloadQuery could be used for listUrgentFlights if desired,
  // but usePaginatedQuery handles the main listing reactively.
  return (
    <Suspense fallback={<FlightsPageSkeleton />}>
      <FlightsClientPage />
    </Suspense>
  );
}

function FlightsPageSkeleton() {
  return (
    <div className="flex flex-col gap-6">
      {/* Filter bar skeleton */}
      <div className="flex gap-3 overflow-x-auto">
        {Array.from({ length: 5 }).map((_, i) => (
          <div key={i} className="h-10 w-32 shrink-0 animate-pulse rounded-lg bg-muted" />
        ))}
      </div>
      {/* Grid skeleton */}
      <div className="grid grid-cols-1 gap-4 md:grid-cols-2">
        {Array.from({ length: 6 }).map((_, i) => (
          <FlightCardSkeleton key={i} />
        ))}
      </div>
    </div>
  );
}
```

The page is an async Server Component that wraps the client page in Suspense. The client page handles all data fetching via Convex hooks.

NOTE: Per research, `preloadQuery` does NOT support pagination. The standard Convex pattern for paginated pages is to let the client-side `usePaginatedQuery` handle data fetching reactively. The Suspense boundary with skeleton provides instant visual feedback.
