  <action>
Create `convex/modules/flights/queries.ts` with three exported query functions:

**1. `listFlights`** — Paginated browse query with multi-field filtering.
- Args: `paginationOpts` (use `paginationOptsValidator` from `convex/server`), `departureCountry` (optional string), `destination` (optional string), `dateFrom` (optional number — Unix timestamp), `dateTo` (optional number), `minSeats` (optional number), `isPackage` (optional boolean).
- Logic:
  - If `departureCountry` is provided, use `by_country_departure` index with `.eq("departureCountry", args.departureCountry)`.
  - Otherwise, use `by_status_departure` index with `.eq("status", "available")`.
  - Apply `filter()` from `convex-helpers/server/filter` (NOT the built-in `.filter()`) for post-index conditions: `!flight.isDeleted`, `flight.approvalStatus === "approved"`, and optional destination/dateFrom/dateTo/minSeats/isPackage checks. When using `by_country_departure`, also filter `flight.status === "available"` in the TypeScript filter since the country index does not constrain status.
  - Call `.paginate(args.paginationOpts)` on the filter result.
  - After pagination, denormalize agent data: for each flight in `result.page`, use `Promise.all()` to fetch `ctx.db.get(flight.agentId)` and merge `agentName`, `agentIsVerified` (agent.isApproved === true), and `agentImageUrl` onto each flight.
  - Return `{ ...result, page: enrichedPage }`.

**CRITICAL:** Use `filter` from `convex-helpers/server/filter` — NOT the built-in Convex `.filter()` before `.paginate()`. The built-in filter produces undersized pages. The convex-helpers filter wraps pagination correctly.

**2. `listUrgentFlights`** — Non-paginated query for flights departing within 24 hours.
- Args: `departureCountry` (optional string).
- Logic:
  - Calculate `now = Date.now()` and `in24h = now + 24 * 60 * 60 * 1000`.
  - Use `by_status_departure` index: `.eq("status", "available").gte("departureDate", now).lte("departureDate", in24h)`.
  - Apply built-in `.filter()` for: `q.eq(q.field("approvalStatus"), "approved")` AND `q.neq(q.field("isDeleted"), true)`.
  - If `departureCountry` provided, add additional `.filter()` for `q.eq(q.field("departureCountry"), args.departureCountry)`.
  - `.order("asc").take(10)` — cap at 10 urgent flights.
  - Denormalize agent data same as listFlights (Promise.all for agent lookups).

**3. `getFlightWithAgent`** — Single flight with full agent info for detail panel.
- Args: `flightId` (v.id("flights")).
- Logic:
  - `ctx.db.get(args.flightId)` — return null if not found or isDeleted.
  - Fetch agent: `ctx.db.get(flight.agentId)`.
  - Return merged object with all flight fields plus `agentName`, `agentIsVerified`, `agentImageUrl`, `agentEmail`.

Import pattern: `import { query } from "../_generated/server";` and `import { v } from "convex/values";` and `import { paginationOptsValidator } from "convex/server";` and `import { filter } from "convex-helpers/server/filter";`.

Note: Do NOT import from `convex/lib/auth.ts` — these are public read queries, no auth needed. Only the mutation needs auth.
