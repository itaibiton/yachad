<objective>
Wire Clerk three-layer authentication: proxy.ts route protection (layer 1), Server Component page guards (layer 2), and Convex function-level auth helpers (layer 3, created in Plan 01-02). Create sign-in/sign-up pages and role-gated route layouts.

Purpose: Ensure every protected route is guarded at multiple layers. Proxy.ts is the first line (UI protection), page guards are the second (SSR protection), and Convex auth helpers are the authoritative gate (API protection). This defense-in-depth is required because middleware alone is bypassable (CVE-2025-29927).

2. Create src/proxy.ts (NOT middleware.ts -- Next.js 16 convention) following RESEARCH.md Pattern 1:

   - Import clerkMiddleware and createRouteMatcher from "@clerk/nextjs/server"
   - Import createMiddleware from "next-intl/middleware"
   - Import routing from "@/i18n/routing" (will exist from Plan 01-04, but reference path now)

   - Create intlMiddleware = createMiddleware(routing)

   - Define route matchers:
     - isPublicRoute: matches /:locale/sign-in(.*), /:locale/sign-up(.*), /:locale (exact home only for public landing)
     - isAgentRoute: matches /:locale/agent(.*)
     - isAdminRoute: matches /:locale/admin(.*)

   - Export default clerkMiddleware handler:
     ```typescript
     export default clerkMiddleware(async (auth, req) => {
       // Run next-intl locale detection/redirect for every request
       const intlResponse = intlMiddleware(req);

       // Protect all non-public routes (require sign-in)
       if (!isPublicRoute(req)) {
         await auth.protect();
       }

       // RBAC: role-based route enforcement
       const { sessionClaims } = await auth();
       const role = sessionClaims?.metadata?.role as string | undefined;

       if (isAgentRoute(req) && role !== "agent" && role !== "admin") {
         return Response.redirect(new URL("/", req.url));
       }
       if (isAdminRoute(req) && role !== "admin") {
         return Response.redirect(new URL("/", req.url));
       }

       return intlResponse;
     });
     ```

   - Export config with matcher that excludes static files, _next internals, and Convex endpoints:
     ```typescript
     export const config = {
       matcher: [
         "/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)",
         "/(api|trpc)(.*)",
       ],
     };
     ```

3. IMPORTANT NOTES:
   - The exported function is the default export (Clerk wraps it). In Next.js 16, `proxy.ts` accepts default export.
   - next-intl's `intlMiddleware(req)` must be called BEFORE auth.protect() fires a redirect, so that locale detection happens on every request.
   - The `auth.protect()` call will redirect unauthenticated users. The redirect URL will include the locale prefix because intlMiddleware already processed the request.
   - If the routing import from "@/i18n/routing" is not yet available (Plan 01-04 creates it), create a temporary stub at i18n/routing.ts:

     ```typescript
     import { defineRouting } from "next-intl/routing";

       locales: ["he", "en"],
       defaultLocale: "he",
       localePrefix: "always",
     });
     ```
     This ensures proxy.ts compiles. Plan 01-04 will use this same file.
