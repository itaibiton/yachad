1. Create convex/lib/auth.ts with three auth helpers following RESEARCH.md Pattern 2:

   **requireUser(ctx: QueryCtx | MutationCtx)**:
   - Call ctx.auth.getUserIdentity()
   - If null, throw "Unauthenticated"
   - Query users table using "by_clerk_id" index with identity.subject
   - If no user found, throw "User not found in database"
   - If user.isBanned, throw "User is banned"
   - Return user document

   **requireAgent(ctx: MutationCtx)**:
   - Call requireUser(ctx) first
   - Check user.role is "agent" or "admin"
   - If role is "agent", ALSO check user.isApproved === true (CRITICAL: DB approval check, not just Clerk role)
   - Throw appropriate errors for missing role or unapproved agent
   - Return user document

   **requireAdmin(ctx: MutationCtx)**:
   - Call requireUser(ctx) first
   - Check user.role === "admin"
   - Throw "Requires admin role" if not
   - Return user document

   Import types from "../_generated/server" for QueryCtx, MutationCtx.

2. Create convex/lib/validators.ts:
   - Export shared Convex validators used across modules
   - Export UserRole type: "user" | "agent" | "admin"
   - Export common validator patterns (e.g., paginationArgs for usePaginatedQuery)

3. Create convex/lib/rateLimit.ts following RESEARCH.md Pattern 8:

     - createPost: token bucket, 10/min, capacity 20

   - Add comment showing usage pattern: `await rateLimiter.limit(ctx, "createPost", { key: userId })`

4. Create convex/http.ts following RESEARCH.md Pattern 4:
   - Import httpRouter from "convex/server"
   - Import httpAction from "./_generated/server"
   - Import Webhook from "svix"
   - Import internal from "./_generated/api"
   - Create httpRouter instance
   - Define POST route at "/clerk-users-webhook":
     - Read CLERK_WEBHOOK_SECRET from process.env
     - Extract svix-id, svix-timestamp, svix-signature headers
     - Read request body as text
     - Verify signature using new Webhook(webhookSecret).verify()
     - If verification fails, return 400 "Invalid webhook signature"
     - Parse event type:
       - "user.created" / "user.updated": call internal.modules.users.mutations.upsertFromClerk with clerkId, email, name, imageUrl, role
       - "user.deleted": call internal.modules.users.mutations.deleteFromClerk with clerkId
     - Return 200 on success
   - Export default http

5. Create convex/modules/users/mutations.ts:
   - Import internalMutation from "../../_generated/server"
   - Import v from "convex/values"

   **upsertFromClerk** (internalMutation -- not exposed publicly):
   - Args: clerkId (string), email (string), name (string), imageUrl (optional string), role (string)
   - Query users by "by_clerk_id" index
   - If exists: patch with updated fields (email, name, imageUrl, role)
   - If not exists: insert new user with role defaulting to "user", isApproved: false

   **deleteFromClerk** (internalMutation):
   - Args: clerkId (string)
   - Query users by "by_clerk_id" index
   - If found: patch with isBanned: true (soft delete, don't remove -- audit trail)

6. IMPORTANT: Use `internal` (not `api`) for mutations called from httpAction. The `internal` namespace ensures these cannot be called from the client.
