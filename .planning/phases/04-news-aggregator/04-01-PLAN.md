**Create `convex/modules/news/queries.ts`** with three exported queries:

1. **`listNewsArticles`** ‚Äî Paginated query for the main news feed.
   - Args: `paginationOpts` (paginationOptsValidator), `language` (optional `v.union(v.literal("he"), v.literal("en"))`), `country` (optional `v.string()`)
   - Use `by_published` index ordered desc (most recent first)
   - Apply `convex-helpers` `filter()` (NOT built-in .filter()) to exclude:
     - `isDeleted === true`
     - `isFeatured === true` (featured articles shown separately)
     - Articles not matching `args.language` when provided
     - Articles not matching `args.country` when provided
   - Denormalize source data via `Promise.all` on `result.page` ‚Äî for each article, fetch its `sourceId` document and merge: `sourceName`, `sourceFaviconUrl`, `sourceTrustTier`
   - Return `{ ...result, page: enrichedPage }`
   - Follow the exact pattern from `convex/modules/flights/queries.ts` listFlights

2. **`listFeaturedArticles`** ‚Äî Non-paginated query for the "Important" pinned section.
   - Args: `country` (optional `v.string()`)
   - Use `by_published` index ordered desc
   - Apply `filter()` to include only `isFeatured === true`, exclude `isDeleted === true`, optionally filter by country
   - `.take(10)` ‚Äî cap at 10 featured articles
   - Denormalize source data same as above
   - Follow the `listUrgentFlights` pattern from flights

3. **`listActiveSources`** ‚Äî Internal query for RSS ingestion action.
   - Export as `internalQuery` (not public query) ‚Äî only called by the internal action
   - Args: none
   - Use `by_active` index where `isActive === true`
   - Return the full source documents (the action needs url, language, name)
   - Import from `"../../_generated/server"` for `internalQuery`

   - Use `by_active_severity` index where `isActive === true`

This file MUST contain ONLY the `internalAction` ‚Äî no queries or mutations (Convex restriction for "use node" files).

1. **`fetchRssFeeds`** ‚Äî `internalAction` that:
   - Fetches all active sources via `ctx.runQuery(internal.modules.news.queries.listActiveSources)`
   - Creates a `new Parser({ timeout: 10_000, headers: { "User-Agent": "Yachad/1.0 RSS Reader" } })`
   - For each source, wraps `parser.parseURL(source.url)` in try/catch ‚Äî log error and `continue` on failure (never let one bad source kill the batch)
   - For each feed item (`.slice(0, 20)` ‚Äî cap at 20 per source):
     - Skip if `!item.link && !item.guid` or `!item.title`
     - Use `item.link ?? item.guid` as the URL (fallback for feeds that use guid instead of link)
     - Use `item.contentSnippet?.slice(0, 200)` for description
     - Use `item.isoDate ? new Date(item.isoDate).getTime() : Date.now()` for publishedAt (fallback for missing pubDate)
     - Set `country: "IL"` for all sources (per research recommendation ‚Äî all Israeli-focused sources)
   - Call `ctx.runMutation(internal.modules.news.mutations.upsertArticles, { articles: articlesToUpsert })` with the batch
   - Import `Parser` from `"rss-parser"` (default export)
   - Import `{ internalAction }` from `"../../_generated/server"` and `{ internal }` from `"../../_generated/api"`
